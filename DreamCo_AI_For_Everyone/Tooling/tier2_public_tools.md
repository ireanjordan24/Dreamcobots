# DreamCo Tier 2 Public / Experimental Tools

## Overview
Tier 2 tools are permitted for use with **public, non-proprietary data only**.
Do NOT enter DreamCo business data, customer data, credentials, or trade secrets
into any Tier 2 tool.

---

## Permitted Tier 2 Tools

| Tool | Permitted Uses | Restrictions |
|------|----------------|-------------|
| ChatGPT (web) | Research, brainstorming | Public info only, no business data |
| Claude.ai (web) | Content drafting, research | Public info only |
| Gemini (web) | Research, summarization | Public info only |
| Hugging Face Hub | Model research, prototyping | No proprietary datasets |
| Public APIs (beta) | Internal PoC and testing | Sandbox/staging only |
| Replit AI | Quick prototyping | No production secrets |
| Cursor AI | Personal coding assistance | No proprietary code |

---

## Tier 2 Rules
1. **Never** enter API keys, passwords, or credentials
2. **Never** enter customer names, emails, or contact details
3. **Never** enter confidential business strategies or financial data
4. **Only** use for personal learning, research, or prototyping with public information
5. **Do not** depend on Tier 2 tools in production workflows
6. **Report** any accidental data entry immediately to the Security Lead

---

## Upgrading a Tool to Tier 1
If a Tier 2 tool proves valuable for production use:
1. Complete a security assessment
2. Submit an approval request (see [tier1_approved.md](tier1_approved.md))
3. Wait for Security Lead sign-off before using with internal data

_See also: [Data Security Policy](../Policies/data_security.md)_
