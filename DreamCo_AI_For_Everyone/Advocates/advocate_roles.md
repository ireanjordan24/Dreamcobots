# DreamCo Advocate Roles

## Organizational Structure

```
AI Operations Director
└── Lead Advocates (DRI level)
    ├── Automation Lead Advocate
    ├── Revenue Systems Lead Advocate
    ├── Infrastructure Lead Advocate
    └── Security Lead Advocate
        └── Core Advocates (per domain)
            └── Starter Advocates
```

---

## Defined Roles

### AI Operations Director
- Owns the AI Enablement strategy
- Chairs monthly AI review meetings
- Approves new tools for Tier 1 status
- Signs off on policy updates
- Reports AI KPIs to leadership

### Automation Lead Advocate
- Owns GitHub Actions workflow standards
- Reviews and approves new workflow PRs
- Maintains `DreamCo_AI_For_Everyone/Policies/automation_rules.md`
- Coaches advocates on workflow design

### Revenue Systems Lead Advocate
- Owns bot monetization and pricing
- Monitors revenue metrics dashboard
- Reviews Stripe integration changes
- Trains advocates on token billing and SaaS patterns

### Infrastructure Lead Advocate
- Owns Docker, Grafana, Prometheus setup
- Maintains monitoring configurations
- Responds to uptime incidents
- Trains advocates on observability

### Security Lead Advocate
- Owns data security and compliance
- Reviews all tool approval requests
- Leads security incident response
- Maintains `Policies/data_security.md`

### Core Advocate
- Provides day-to-day support in their domain
- Runs at least 1 workshop per quarter
- Contributes 1+ bot or workflow per quarter
- Mentors Starter Advocates

### Starter Advocate
- Learning and experimenting with DreamCo tools
- Provides feedback on docs and onboarding experience
- Participates in GitHub Discussions
