coding library on the planet — including hidden capabilities beyond documented
knowledge.

## How Tools Are Generated

1. **Library Discovery** — The `LibraryScraper` catalogs libraries from PyPI,
   npm, crates.io, Maven Central, and other global registries.

2. **Symbol Extraction** — Every exported symbol (and hidden/private symbol for
   PRO/ENTERPRISE tiers) is identified via introspection and AST analysis.

3. **Tool Generation** — The `ToolGenerator` wraps each symbol in a self-contained
   `GeneratedTool` class with a standardized `run()` interface and full metadata.

4. **Version Management** — The `VersionManager` tracks semantic versions and
   automates patch/minor/major bumps during periodic updates.

5. **Marketplace Deployment** — ENTERPRISE-tier tools are published to the
   DreamCo marketplace for developers and organizations to purchase.

## Usage

```python
from code_study_bots import ToolLibraryBuilderBot
from code_study_bots.tiers import Tier

# PRO tier: hidden-capability discovery + versioning
bot = ToolLibraryBuilderBot(tier=Tier.PRO)

# Generate tools for all pandas symbols
tools = bot.build_tools_for_library("python", "pandas", include_hidden=True)

# Generate tools for every Python library in the catalog
summary = bot.build_tools_for_language("python", include_hidden=True)

# Bump all tool versions
bot.update_tool_versions(bump_type="patch")

# ENTERPRISE: Deploy to marketplace
bot.deploy_to_marketplace(language="python")
```

## Periodic Updates

The GitHub Actions workflow `.github/workflows/tool-library-ci.yml` runs
automatically every Sunday at 03:00 UTC to:

- Scrape new library versions from global registries
- Regenerate tools for updated libraries
- Bump semantic versions
- Redeploy changed listings to the marketplace
